Prerequisites  

Before you begin, you should have:

- A GitHub account and access to the `esinchq-cloud` organization  

- Access to the ESINC Agentic OS repositories or starter templates  

- At least one content platform account to test with, such as YouTube, Instagram, TikTok, or Threads  

- Access to an AI model or provider that will power your agents  

- A basic automation environment, such as n8n or another workflow runner  

Get started  

1. Choose your starting system  

Pick the main ESINC repo, workflow, or orchestrator you want to begin with. Start with the smallest useful version of the system rather than trying to activate everything at once.  

2. Configure your environment  

Set up your environment variables, API keys, model access, and platform credentials. Begin with one AI provider and one publishing destination so testing stays simple.  

3. Run a first test workflow  

Launch the local system or workflow and test one basic task, such as generating a content idea, creating a caption, or drafting a post for one platform.  

4. Connect one real publishing path  

Link the system to one real platform through n8n, MCP, or another automation layer. The goal is to prove one end-to-end flow from idea to published asset.  

5. Review and improve  

Document what worked, what broke, and what still required manual effort. Use those notes to improve the next version of the system and reduce human intervention over time.  

Need help?  

Use this documentation as the source of truth for setup, workflows, tools, and architecture. As the system evolves, this quickstart should become the simplest path from zero to first successful automation.
